销售 / 战略运营

想要一套「更好用的智能 IT 系统」

把被 CRM 录入、合同流转、翻译校对、文档生成这些流程困住的时间释放出来。AI 是助手，负责把已经存在的流程跑得更轻。

"理想状态：通过自然语言指令由 AI 自动完成 CRM 更新 / 翻译 / 校对。"
—— sales lead 访谈
产研 / 售前售后（领导层）

想要一支「AI Agent 开发 & 交付团队」

自主接需求、自主完成项目、像人类研发团队一样交付产物；或在客户现场无需研发支持独立完成部署调试。AI 是生产要素，要替人扛交付。

"让 PM Agent 自动拆解，协调其他角色 Agent 协作 …… 完整代码 + 部署验证 + 齐全文档。"
—— 产品访谈
Agent 开发团队在SensePedia Agent Platform项目中已经探索出来了一些经验，它的主体属性——和工业革命的汽车、信息革命的电脑一样——仍然是工具。
不能对着 Agent 许个愿，它就自动交付。得有人驱动。
Agent 相对于人，缺的三样东西：
  • 主动思考 & 提问（不会停下来问"这个需求是不是有问题"）
  • 长程任务规划（多轮后目标漂移，从主线滑向细节）
  • 偏差识别与纠偏（错误输入 → 错误输出 → 下一轮输入，一夜空转）
↑ SensePedia Agent Platform 开发过程已经踩过这个坑。
