Technical Overview

Spatial Agent Memory 与 SpatialRAG

这是一套面向机器人与空间智能体的三维记忆与检索框架。关键不只是“看见世界”，而是把原始感知转化为可持续更新、可检索、可推理的空间知识系统。

点云负责感知 保留原始三维观测，提供高保真几何证据。
体素负责记忆 把空间离散化为统一网格，承载语义、时间和索引。
SpatialRAG 负责调用 在任务发生时检索相关空间上下文，支撑回答、导航与行动决策。

1. 什么是体素（Voxel）？

体素是三维空间中的基础表达单元，本质上是像素在空间维度上的延展。

像素（Pixel）是二维图像的最小单位，体素（Voxel）= Volume + Pixel，可以理解为像素在三维空间中的对应物。

如果把连续的三维空间切成大量规则的小立方体，那么每个小立方体就是一个体素。每个体素对应一个固定空间位置，并可携带丰富属性。

空间表达

体素的核心价值在于把连续世界离散化、结构化，形成适合计算、融合和检索的三维网格。

可挂载信息
  • 占据状态与可通行性
  • 颜色、纹理、材质特征
  • 语义标签与向量嵌入
  • 观测时间与更新状态

2. 什么是点云（Point Cloud）？

点云更接近传感器的原始观测结果，是机器人感知空间的第一手几何证据。

点云通常由激光雷达（LiDAR）、深度相机或其他三维传感器产生，本质上是大量三维坐标点的集合。

每个点至少包含三维位置 (x, y, z)，有时还会附带反射强度、颜色、法向量等属性。点云更接近传感器对世界的原始观测结果。

它像一张三维草图，能清晰反映轮廓和几何形状，但通常只有“点”，没有天然的体积填充与规则网格结构。

典型特征：稀疏、无序、密度不均匀，受距离、视角和遮挡影响明显。它保真度高，但直接用于长期记忆和检索并不理想。

3. 点云和体素的区别

两者都能表达三维空间，但属于不同层级的表示方法。前者偏原始观测，后者偏结构化建模。

维度 点云（Point Cloud） 体素（Voxel）
本质 原始三维采样点集合 规则化的三维空间网格
数据形态 无序、稀疏、不规则 有序、离散、结构化
来源 直接来自传感器扫描 由点云、图像和位姿信息融合后构建
表达重点 哪里测到了点 这个空间单元里有什么
空间覆盖 主要记录观测到的表面点 可表达占据、空闲、未知等状态
计算特点 保真度高，但处理复杂 更适合融合、索引、检索和推理
存储特点 大场景下通常更紧凑 分辨率越高，存储成本越大
典型用途 重建、测量、目标检测、几何匹配 语义地图、空间记忆、规划与 SpatialRAG
一句话理解

点云是机器人“看到”的原始三维世界。

体素是机器人“整理和记住”的结构化三维世界。

工程关系

二者通常不是替代关系，而是上下游关系。常见流程是先采集点云，再进行融合和体素化，构建长期可用的空间记忆。

4. 什么是 Spatial Agent Memory？

核心目标不是保存零散传感器数据，而是形成一个持续更新、可查询、带语义的三维世界模型。

Spatial Agent Memory 是面向机器人或空间智能体的多模态三维记忆系统。它把图像、激光雷达、位姿信息与点云统一融合到体素化空间中，形成长期可维护的空间记忆。

数据来源 作用
视频帧 / RGB 图像 提供外观、颜色、纹理信息
激光雷达 / 深度传感器 提供高精度三维几何结构
里程计 / SLAM 位姿 提供机器人位置、视角和运动轨迹
点云 提供场景的原始空间采样与几何证据

体素化之后，系统能在每个空间单元上挂载更丰富的信息：

  • 占据状态：该位置是否有物体，是否可通行
  • 语义标签：例如门、桌子、椅子、墙面
  • 视觉特征：颜色、纹理、图像嵌入
  • 几何特征：形状描述、局部结构信息
  • 时间信息：何时首次观测、最近更新时间
  • 向量嵌入：便于相似性检索与任务推理
工程上的关键取舍：点云适合感知，体素适合记忆。Spatial Agent Memory 的价值就在于把多源观测沉淀为统一的空间知识单元。

5. 什么是 SpatialRAG？

SpatialRAG 把“检索增强生成”的思路，从文本知识库扩展到三维空间记忆。

RAG（Retrieval-Augmented Generation）本质上是“先检索，再生成”。SpatialRAG 则将这种逻辑应用到机器人空间系统中：任务发生时，机器人先在空间记忆中检索与当前目标最相关的上下文，再结合推理模型输出回答或行动决策。

输入问题或任务
    ↓
在空间记忆中进行多维检索
    ↓
找到相关空间上下文
    ↓
将上下文与任务一起输入推理模型
    ↓
输出有空间依据的回答或行动决策

SpatialRAG 的检索可以跨多个维度同时进行：

  • Object：目标物体在哪里，例如“红色杯子在哪”
  • Room：特定房间内有哪些物体与结构
  • Semantic：语义上适合某类行为的位置，例如“适合坐下休息的地方”
  • Geometry：是否有足够高度、宽度和净空供机器人通行
  • Time：某个物体最近一次出现的时间
  • Image：外观相似区域或相似物体
  • Point Cloud / Shape：几何结构或局部形状相似的区域

其中，点云维度主要提供原始几何匹配依据，体素化记忆则提供统一坐标系下的长期索引基础。二者结合，使检索结果既有几何证据，又能被推理系统直接消费。

6. 整体架构

从物理世界到推理决策，这套系统的主链路可以概括为感知、建模、索引、检索和推理五个阶段。

物理世界
    ↓ 传感器采集
图像 / 视频 + 激光雷达 + 深度数据 + 里程计
    ↓ 数据配准与融合
点云构建与更新
    ↓ 体素化表示
Spatial Agent Memory（三维语义记忆）
    ↓ 多维向量索引
SpatialRAG 检索
    ↓ 上下文增强推理
机器人回答、导航与行动决策
建模阶段

将多模态观测统一对齐到同一空间坐标系，并持续写入体素记忆。

调用阶段

在任务触发时，从体素记忆和附属向量索引中检索最相关的空间上下文，供模型决策使用。

7. 总结

点云负责感知世界，体素负责组织世界，Spatial Agent Memory 负责记住世界，SpatialRAG 负责在需要时找出相关空间知识。

相比传统机器人依赖瞬时感知或扁平地图的方式，这套系统提供的是一种持久化、语义化、可检索、可推理的空间知识底座，使机器人不只是“看见环境”，而是具备对环境的长期空间认知能力。
