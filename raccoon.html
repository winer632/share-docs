Part 1 — 小浣熊 Skill 技术形态
01

raccoon-dataanalysis-skill 架构解析

小浣熊团队已在 ClawHub（OpenClaw 的技能市场）上发布了 raccoon-dataanalysis-skill（v1.0.3），提供数据分析、文件处理、可视化等能力。以下是其技术形态的完整分析。

🦝
Skill 包组成与核心能力
  • Skill 包组成：SKILL.md（描述 + 触发词 + 行为规则）+ Python 脚本（RaccoonClient 类）+ API 参考文档 + 速查表
  • 环境变量：RACCOON_API_HOST（服务地址）+ RACCOON_API_TOKEN（Bearer Token 认证）
  • 通信方式：HTTPS REST API + SSE 流式响应，支持会话管理、文件上传、多轮对话
  • 执行模式：所有数据分析在小浣熊远端沙箱执行（不在本地执行），内置 3 次自动重试
  • 核心能力：数据分析会话管理、文件上传/下载（7 天过期）、代码生成与执行、数据可视化（图表生成）、OCR 识别

Part 2 — SensePedia 对接路径
02

三种可行的对接方案

根据 SensePedia Agent Platform 的架构设计，我们有三种路径可以对接小浣熊的能力，适用于不同阶段和需求。

A
Skill 原生注册（推荐）
  • 将小浣熊能力注册到 SensePedia 的 Skill Registry
  • Skill Loader 加载描述元数据
  • Skill Executor 调用小浣熊 REST API
  • 优势：最直接，改动最小，快速落地
  • 适用：MVP 阶段快速验证
B
MCP Server 封装
  • 将小浣熊 API 封装为标准 MCP Server
  • 通过 SensePedia 的 MCP Adapter 接入
  • Schema 映射 + 权限注入自动化
  • 优势：标准化协议，可扩展更多能力
  • 适用：长期架构演进
C
HostOps API 直接集成
  • 在 HostOps 层直接调用小浣熊 REST API
  • 支持文件预处理、结果后处理
  • 完全控制调用时序和错误处理
  • 优势：深度定制，最大灵活性
  • 适用：需要深度定制的高级场景
Part 3 — 端到端调用示例

飞书数据分析场景

场景：客户在飞书群里问"帮我分析这份销售数据，画一个趋势图"

飞书用户
→
Channel Adapter
→
Agent Router
→
RAG Agent
→
小浣熊 Skill
→
图表返回
步骤 组件 动作
1 飞书 Bot → Channel Adapter 接收消息，提取用户身份，解析附件
2 API Gateway → Agent Router 认证用户，识别为数据分析任务，路由到 RAG Agent
3 RAG Agent → Skill Executor LLM Planner 决策调用小浣熊 Skill
4 Skill Executor → 小浣熊 API 创建会话 → 上传文件 → 发送分析请求 → SSE 流式接收结果
5 小浣熊 API → Skill Executor 返回分析文本 + 图表（PNG/SVG），下载生成物
6 Channel Adapter → 飞书 Bot 将图表和分析结论格式化后推送回飞书群
关键点：SensePedia 负责编排和渠道管理，小浣熊负责数据分析执行。各司其职，通过标准化接口协作。RACCOON_API_HOST 可配置为国内部署地址，确保数据不出境。
